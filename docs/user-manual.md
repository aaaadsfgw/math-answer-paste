# Math Answer Paste 利用説明書

## 1. このアプリについて

Math Answer Paste は、ブラウザ上で選択した数学・情報系の問題文を、ローカルPC上の Ollama に送信し、答えだけを1行でクリップボードに保存する Chrome 拡張機能です。

通常の使い方では、問題文を選択して `Ctrl+Shift+Y` を押すだけで解析が始まり、完了後に `Ctrl+V` で回答を貼り付けられます。

クラウドAI APIは使わず、PC内で動作する Ollama のローカルAPIを利用します。そのため、OpenAI APIキーなどの外部APIキーは不要です。

## 2. 主な機能

- 問題文の手動入力
- Webページ上の選択テキスト読み込み
- `Ctrl+Shift+Y` によるショートカット実行
- Ollama local API への送信
- 答えだけ1行の抽出
- クリップボードへの自動コピー
- ページ右上の小型トースト通知
- 回答履歴の保存・削除
- モデル名、API URL、デモモードなどの設定保存
- Ollamaが使えない環境向けのデモモード

## 3. 動作に必要なもの

この拡張を通常モードで使うには、以下が必要です。

- Google Chrome
- Ollama
- `qwen3:8b` などのローカルLLMモデル
- このリポジトリのファイル一式

推奨モデルは `qwen3:8b` です。実機テストでは、回答速度と精度のバランスが良く、`/no_think` を付けることで比較的短時間で回答できました。

## 4. Ollamaのインストール方法

### 4.1 公式サイトからインストールする場合

Ollama公式サイトからインストーラをダウンロードします。

```text
公式ダウンロードページ:
https://ollama.com/download

Windows用ダウンロードページ:
https://ollama.com/download/windows
```

Windowsの場合は、ダウンロードした `OllamaSetup.exe` を実行してインストールします。

インストール後、PowerShell または コマンドプロンプトで次を実行します。

```bash
ollama --version
```

バージョン番号が表示されれば、Ollamaのインストールは完了です。

### 4.2 PowerShellコマンドでインストールする場合

Windowsでは、PowerShellから次のコマンドでインストールする方法もあります。

```powershell
irm https://ollama.com/install.ps1 | iex
```

学校や共有PCなどで実行ポリシーや権限により失敗する場合は、公式サイトからインストーラをダウンロードして実行してください。

### 4.3 macOS / Linuxの場合

macOSやLinuxでもOllamaは利用できます。環境がWindows以外の場合は、公式ダウンロードページから対象OSを選びます。

```text
https://ollama.com/download
```

この説明書では、主にWindows環境での利用を前提に説明します。

## 5. Ollamaの準備

Ollamaをインストールした後、ターミナルまたはPowerShellで次を実行します。

```bash
ollama pull qwen3:8b
ollama run qwen3:8b
```

`ollama pull qwen3:8b` は、使用するモデルをPCにダウンロードするコマンドです。初回は時間がかかります。

`ollama run qwen3:8b` を実行すると、モデルが起動します。入力待ち状態になっていれば準備完了です。

Ollamaが起動しているか確認する場合は、別のターミナルで次を実行します。

```bash
ollama ps
```

`qwen3:8b` が表示されていれば、モデルは起動しています。

## 6. Chrome拡張として読み込む方法

1. Chromeで `chrome://extensions/` を開きます。
2. 右上の「デベロッパーモード」をONにします。
3. 「パッケージ化されていない拡張機能を読み込む」を押します。
4. `manifest.json` が入っている `math-answer-paste` フォルダを選択します。
5. 拡張機能一覧に Math Answer Paste が表示されれば読み込み完了です。

読み込むフォルダは、次のような構成になっている場所です。

```text
math-answer-paste/
  manifest.json
  popup.html
  history.html
  settings.html
  examples.html
  help.html
  about.html
  js/
  css/
  docs/
```

## 7. 初期設定

拡張機能の設定画面で、以下を確認します。

```text
modelName: qwen3:8b
apiUrl: http://localhost:11434/api/chat
defaultMode: answer
demoMode: OFF
saveHistory: ON
```

通常のAI回答を使う場合は、`demoMode` をOFFにします。ONのままだと、固定されたデモ問題だけをローカルJavaScriptで処理します。

## 8. 基本の使い方：popupから解く

1. Chromeの拡張アイコンから Math Answer Paste を開きます。
2. 問題文を入力欄に貼り付けます。
3. 出力モードを選びます。
4. 「解く」を押します。
5. 回答が表示されたら、コピーして利用します。

Ollamaとの接続確認をする場合は、まずこの方法で `2x + 3 = 11` のような簡単な問題を試すと分かりやすいです。

## 9. ショートカットで解く方法

1. Webページ上の問題文をドラッグして選択します。
2. `Ctrl+Shift+Y` を押します。
3. ページ右上に小さく「解析中...」と表示されます。
4. 解析が終わると「コピー完了: 回答」と表示されます。
5. 回答はクリップボードに入っているので、必要な場所で `Ctrl+V` を押して貼り付けます。

この機能は、通常のコピー操作に完全に割り込むものではありません。Chrome拡張専用のショートカットとして動作します。

`Ctrl+Shift+C` はChrome DevToolsなどと衝突しやすいため、標準ショートカットは `Ctrl+Shift+Y` にしています。

## 10. トースト通知について

ショートカット実行時には、ページ右上に小さな通知が表示されます。

- `解析中...`: 問題文を取得し、Ollamaに送信しています。
- `コピー完了: 回答`: 回答をクリップボードに保存しました。
- `エラー: ...`: 問題文取得、Ollama接続、クリップボード書き込みなどに失敗しました。

通知はページ閲覧を邪魔しないよう、小さめに表示し、数秒後に自動で消えます。

## 11. ショートカットが動かない場合

Chromeで次を開きます。

```text
chrome://extensions/shortcuts
```

Math Answer Paste のショートカットが `Ctrl+Shift+Y` に設定されているか確認します。

他の拡張機能やブラウザ機能と衝突している場合は、別のキーに変更してください。

## 12. 出力モード

設定やpopupで出力モードを切り替えられます。

- `answer`: 答えだけを出します。
- `steps`: 途中式を含めます。
- `explain`: 簡単な解説を含めます。
- `hint`: ヒントだけを出します。

ショートカット実行時は、素早く貼り付ける用途を優先し、基本的に `answer` モードで処理します。

## 13. 出力例

```text
問題: 2x + 3 = 11
回答: x=4
```

```text
問題: 1011(2)を10進数に変換した値はどれか。ア 9 イ 10 ウ 11 エ 12
回答: ウ
```

```text
問題: 赤玉3個、青玉2個が入った袋から同時に2個取り出す。2個が同じ色である確率を求めよ。
回答: 2/5
```

```text
問題: a_1=2, a_{n+1}=3a_n+1 で定義される数列の一般項 a_n を求めよ。
回答: (5*3^n-3)/6
```

## 14. 履歴機能

回答結果は `chrome.storage.local` に保存されます。

履歴画面では、過去に解いた問題文、回答、モード、実行時刻を確認できます。不要になった履歴は削除できます。

履歴保存を行いたくない場合は、設定画面で履歴保存をOFFにできます。

## 15. デモモード

デモモードは、Ollamaが起動していなくても固定例だけを試せる機能です。

授業中や発表時にOllamaが動かない場合でも、最低限の画面動作を見せるために用意しています。

ただし、デモモードは固定パターンにしか対応しません。実際のAI回答を使う場合は、デモモードをOFFにしてください。

## 16. データの保存場所

設定と履歴は Chrome の `chrome.storage.local` に保存します。

ブラウザ外でHTMLを直接開いた場合の確認用として、一部処理は `localStorage` にフォールバックします。

保存する主な情報は以下です。

- モデル名
- Ollama API URL
- デフォルト出力モード
- デモモード設定
- 履歴保存設定
- 回答履歴

## 17. プライバシーとAPIキーについて

この拡張は、クラウドAI APIを使用しません。

問題文は、ローカルPC上で動いている Ollama API に送信されます。API URLは標準で `http://localhost:11434/api/chat` です。

OpenAI APIキーなどの外部APIキーは使わず、リポジトリにも保存しません。

## 18. できないこと・制限

この拡張には次の制限があります。

- OllamaをChrome拡張だけで自動起動することはできません。
- 使用前にOllamaまたは対象モデルを起動しておく必要があります。
- AIの回答は必ず正しいとは限りません。
- Chrome内部ページ、PDFビューア、Google Docs、一部の学校システムでは選択テキスト取得に失敗する場合があります。
- ページによってはクリップボード書き込みが制限される場合があります。
- 長文問題や画像問題には向いていません。

Chrome拡張はセキュリティ上、PC上の任意のアプリやコマンドを直接起動できません。そのため、Ollamaの起動はユーザーが事前に行う設計にしています。

## 19. よくあるエラーと対処

### 接続できない

Ollamaが起動していない可能性があります。

```bash
ollama run qwen3:8b
```

または、Ollamaの状態を確認します。

```bash
ollama ps
```

### モデルが見つからない

指定したモデルがPCに入っていない可能性があります。

```bash
ollama pull qwen3:8b
```

### 回答が遅い

モデルが重い、またはGPU/CPUの処理が詰まっている可能性があります。

このプロジェクトでは、速度と精度のバランスから `qwen3:8b` を第一候補にしています。

### ショートカットを押しても何も起きない

以下を確認します。

1. `chrome://extensions/shortcuts` でショートカットが設定されているか
2. 問題文を選択しているか
3. Chrome内部ページなど、拡張が動けないページで試していないか
4. 拡張機能を再読み込みしたか

### 回答が貼り付けられない

回答は自動でクリップボードに入ります。自動で入力欄に挿入されるわけではありません。

必要な場所で `Ctrl+V` を押して貼り付けます。

## 20. 発表時の説明例

この作品は、Chrome拡張とローカルLLMを組み合わせた学習支援ツールです。

ブラウザ上の問題文を選択し、`Ctrl+Shift+Y` を押すと、選択テキストをOllamaのローカルAPIへ送信します。AIの回答から答えだけを抽出し、クリップボードへ保存します。ユーザーはそのまま `Ctrl+V` で回答を貼り付けられます。

クラウドAPIを使わず、ローカルPC上でAIを動かすため、APIキー管理や外部送信の問題を避けられる点が特徴です。

また、処理中・成功・失敗が分かるように、ページ右上に小型トースト通知を表示しています。通常の閲覧を邪魔しないように、小さく数秒だけ表示される設計にしました。

## 21. 開発時に重視した点

- Chrome拡張として実際に読み込めること
- ショートカット操作で素早く使えること
- 回答を自動でクリップボードに入れること
- 外部APIキーを使わないこと
- OllamaとローカルLLMの導入手順を説明できること
- AIの応答から答えだけを抽出すること
- 処理状態が分かるようにトースト通知を追加すること

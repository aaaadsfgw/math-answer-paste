# Math Answer Paste

Math Answer Paste は、数学・情報系の問題文をローカルPC上の Ollama に送り、答えだけを1行で表示・コピーできる Chrome 拡張機能です。

## 概要

- 問題文を入力、またはブラウザ上の選択テキストから読み込みます。
- `Ctrl+Shift+Y` で、選択中の問題文を解いて最終回答だけをクリップボードへ入れられます。
- ショートカット実行時は、ページ右上に小さなトーストで「解析中」「コピー完了」「エラー」を表示します。
- ローカルの Ollama API `http://localhost:11434/api/chat` にだけ送信します。
- OpenAI API や外部APIキーは使いません。
- AIの返答から `<think>...</think>` を除去し、最後の空でない1行を最終回答として扱います。
- Ollama が使えない環境でも試せるデモモードを用意しています。

## 詳細説明書

利用手順、ショートカット、トースト通知、Ollama設定、エラー対処、発表時の説明例は [`docs/user-manual.md`](docs/user-manual.md) にまとめています。

## 作成目的

提出課題や学習用途で、計算問題・情報系の基本問題の答えをすばやくコピーできるようにするための土台です。最終仕様では角括弧を付けず、`x=4` や `ウ` のような答え単体を表示します。

## Chrome拡張の読み込み方法

1. Chromeで `chrome://extensions/` を開きます。
2. 右上の「デベロッパーモード」をONにします。
3. 「パッケージ化されていない拡張機能を読み込む」を押します。
4. この `math-answer-paste` フォルダを選びます。

## Ollamaのインストールと起動方法

Ollama未導入の場合は、公式サイトからインストールします。

```text
公式ダウンロードページ:
https://ollama.com/download

Windows用ダウンロードページ:
https://ollama.com/download/windows
```

Windowsでは、公式サイトから `OllamaSetup.exe` をダウンロードして実行する方法が一番分かりやすいです。

インストールできたか確認するには、PowerShell またはコマンドプロンプトで次を実行します。

```bash
ollama --version
```

バージョン番号が表示されればインストール完了です。

その後、使用するモデルをダウンロードして起動します。

```bash
ollama pull qwen3:8b
ollama run qwen3:8b
```

`ollama run qwen3:8b` を実行し、入力待ち状態になっていれば準備完了です。状態を確認する場合は別のターミナルで `ollama ps` を実行します。

## 推奨モデルと設定

1. `qwen3:8b`
   - RTX 4060 VRAM 8GB 環境で、回答精度と速度のバランスが良かったため第一候補です。
   - `/no_think` を user message の先頭に付けることで応答速度の改善が見られたため、標準で使います。
2. `deepseek-r1:14b`
   - 精度重視の候補です。正答できる一方、CPU/GPU混在でやや重くなる可能性があります。
3. `deepseek-r1:8b`
   - 軽量候補ですが、環境によってはGPU 100%でも回答が遅い場合があります。

## ショートカット機能

1. Webページ上の問題文を選択します。
2. `Ctrl+Shift+Y` を押します。
3. ページ右上に小さく「解析中...」と表示し、拡張機能が選択テキストを取得して Ollama に送信します。
4. 完了すると「コピー完了: 答え」と表示し、最終回答だけをクリップボードに入れます。
5. ユーザーは `Ctrl+V` で任意の場所に貼り付けます。

通常のコピー操作に完全に割り込むのではなく、拡張機能専用のショートカットとして動作します。`Ctrl+Shift+C` はChrome DevTools等と衝突しやすいため、標準ショートカットは `Ctrl+Shift+Y` にしています。ショートカットが他の機能と衝突する場合は、`chrome://extensions/shortcuts` から変更してください。

トースト表示はページ右上に小さく出る補助表示です。通常のページ閲覧を邪魔しないよう、数秒後に自動で消えます。

## Ollama連携の仕組み

`js/ollama-client.js` が `fetch` で `POST /api/chat` を呼び出します。APIには過去の会話履歴を送らず、毎回その問題だけを単発で処理します。CLIの `/clear` 相当として、`messages` には現在の `system` と `user` だけを入れます。

user message の先頭には `/no_think` を付けます。

```text
/no_think
問題：2x + 3 = 11
```

## 出力例

- `2x + 3 = 11` -> `x=4`
- `1011(2)を10進数に変換` -> `11`
- `800円の25%` -> `200`
- `1011(2)` と選択肢 `ア 9 イ 10 ウ 11 エ 12` -> `ウ`
- 不明な場合 -> `不明`

## データ保存方法

設定と履歴は `chrome.storage.local` に保存します。ブラウザ外でHTMLを直接開いた場合の確認用に、一部処理は `localStorage` にフォールバックします。

## デモモードについて

Ollamaが起動していなくても、固定例はローカルJavaScriptだけで回答します。デモモードでも角括弧は付けません。

## よくあるエラー

- 接続できない: `ollama run qwen3:8b` を実行し、Ollamaが起動しているか確認してください。
- モデルがない: `ollama pull qwen3:8b` を実行してください。
- 返答が遅い: `qwen3:8b` と `/no_think` の組み合わせを第一候補として試してください。
- `Ctrl+Shift+Y` が反応しない: `chrome://extensions/shortcuts` でショートカット割り当てを確認してください。
- 特定ページで選択テキストを取得できない: popup画面に手動で貼り付けて実行してください。

## AI活用ポイント

この拡張はクラウドAIではなくローカルLLMを使います。問題文を外部送信しない設計にし、APIキーや秘密情報をリポジトリへ置かない方針です。

Chrome拡張をユーザーインターフェース、Ollama local API をAI処理ツールとして扱い、ショートカット操作からAI回答を生成する構成にしています。
